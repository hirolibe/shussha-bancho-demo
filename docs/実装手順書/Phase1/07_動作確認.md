実機にビルドを焼いて、Phase 1 の **受け入れ基準** をひとつずつ潰す。

## 7.1 ビルド方法の選択

| 方式 | メリット | デメリット | 推奨ケース |
|---|---|---|---|
| `npx expo run:ios --device` | 最速。Xcode の Signing がそのまま使える | Mac 必須、有線接続必要 | iOS 個人検証 |
| `npx expo run:android --device` | 最速。adb 経由 | USB デバッグ ON 必要 | Android 個人検証 |
| EAS Build (`eas build --profile development`) | クラウドビルド。Xcode 不要 | ビルド待ち 10〜30分 | iOS で証明書を会社共有したい場合 |

```bash

```

## 7.1.A 初回 iOS 実機ビルドの完全ガイド

ここを順番に通せば、無料 Apple ID (Personal Team) で iPhone に焼いて起動できる。
**実機が初めて参加するセッションでは必ず 1 回通す**。

### A-1. Xcode のインストールと選択

Xcode (フル版・Mac App Store) が必要。Command Line Tools 単体では `xcrun --sdk iphoneos` が失敗するため不可。

```bash
# 状態確認
xcode-select -p
# → /Library/Developer/CommandLineTools が出たらフル版未選択

# 確認: Xcode.app の在処
ls /Applications | grep -i Xcode

# Xcode を CLT より優先させる
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license accept

# 動作確認 (パスが返れば OK)
xcrun --sdk iphoneos --show-sdk-path
```

### A-2. iOS Platform Support のダウンロード

接続予定の iPhone の iOS バージョンに対応するプラットフォーム支援パッケージが Xcode に必要。
Xcode 起動 → メニュー `Xcode > Settings... > Components` で対象 iOS を `Get`。
**1 つあたり 5〜10 GB** あるので空き容量が必要。

#### ディスク容量が不足したとき

```bash
# 安全に削れるもの (合計 7+ GB が現実的)
rm -rf ~/Library/Developer/Xcode/DerivedData          # 1〜2 GB / 完全に再生成可
rm -rf ~/Library/Developer/Xcode/iOS\ DeviceSupport   # 5+ GB / 次回接続時に再生成 (数分)
rm -rf ~/Library/Caches/CocoaPods                     # 数百 MB
df -h /System/Volumes/Data | tail -1                  # 残量再確認
```

> Chrome キャッシュ等の他アプリのキャッシュも削れるが、本流ではないので Xcode 系を先に。

### A-3. Apple ID の Xcode 登録 (Personal Team)

無料の Apple ID で OK。年 $99 の Apple Developer Program は不要。

- Xcode メニュー `Xcode > Settings... > Accounts > +` → Apple ID
- 登録後 `(Personal Team)` 表示が出れば OK

> Personal Team は **7 日間有効の開発ビルド**。期限が切れたら再ビルドで延命。

### A-4. prebuild → pod install

`react-native-beacon-radar` を含めた native コードを iOS プロジェクトに展開する。

```bash
# 既存の ios/ android/ を一度クリアして再生成
rm -rf ios android
npx expo prebuild --clean
```

prebuild は内部で:

1. `app.json` をもとに `ios/<Project>` ディレクトリを生成 (`expo.name` の ASCII 部分が Xcode プロジェクト名)
2. `ios/Podfile` を生成
3. `pod install --repo-update` を実行

**ここで失敗するパターンと対処** は §7.5 を参照。代表的なのは `RCT-Folly` 解決失敗 (新アーキ問題) と `xcrun: SDK iphoneos cannot be located` (xcode-select 未切替)。

### A-5. Xcode で Workspace を開く

```bash
open ios/ShusshaBancho.xcworkspace
```

> `.xcworkspace` を開くこと（`.xcodeproj` ではなく）。CocoaPods を使うときは workspace が正解。
> プロジェクト名 (`ShusshaBancho`) は `app.json` の `expo.name` に依存する。

### A-6. Signing & Capabilities の設定

1. 左ペイン最上部の **青いアイコン** (プロジェクト) をクリック
2. 中央エリアで `TARGETS > ShusshaBancho`
3. 上部タブ **`Signing & Capabilities`**
4. `Automatically manage signing` ✅
5. `Team` プルダウンから `〜さん (Personal Team)` を選択
6. もし Bundle Identifier 重複エラーが出たら、お尻に何か追加して一意化:
   `jp.co.fusic.shusshabancho.phase1` → `jp.co.fusic.shusshabancho.phase1.tagami`
7. 「Status」欄が緑になればOK

#### Push Notifications エラーが出た場合

```
Cannot create a iOS App Development provisioning profile for ...
Personal development teams ... do not support the Push Notifications capability.
```

`expo-notifications` plugin が `aps-environment` entitlement を不可避で追加するのが原因。

```bash
# 1. app.json から expo-notifications を plugins から外す (Step 1.3 で実施済)
# 2. 既存の entitlements ファイルから aps-environment を消す
```

`ios/ShusshaBancho/ShusshaBancho.entitlements` を以下のように空にする:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
  <dict>
  </dict>
</plist>
```

Xcode の `Signing & Capabilities` タブで「Try Again」をクリック。Status が緑になればOK。

### A-7. iPhone 側の準備

- iPhone を Mac に **Lightning / USB-C ケーブルで有線接続**
- iPhone のロック解除
- 「**このコンピュータを信頼しますか？**」が出たら **「信頼」**
- iPhone の **設定 > プライバシーとセキュリティ > デベロッパモード** を ON (iOS 16+)
  - ON にすると再起動が要求される。再起動後、ロック解除時に確認モーダルが出るので「ON にする」

確認:

```bash
xcrun devicectl list devices
# 自分の iPhone が表示されればOK
```

### A-8. ビルド & 起動

Xcode 上部のデバイス選択 (再生ボタンの右) で接続中の iPhone を選択 → **▶ ボタン**。
または:

```bash
npx expo run:ios --device
```

初回ビルドは 5〜15 分。完了するとアプリが iPhone にインストールされ、自動起動を試みる。

#### 初回起動時の "Untrusted Developer"

1. iPhone のホーム画面でアプリアイコンをタップしようとすると「信頼されていないデベロッパ」と表示
2. iPhone の **設定 > 一般 > VPN とデバイス管理**
3. `Apple Development: <自分のメール>` をタップ
4. 「**信頼**」をタップ
5. ホーム画面に戻ってアイコンタップ → 起動

### A-9. アプリ起動後の確認

1. 設定画面で Slack トークン (xoxp-...) と Beacon UUID を保存
2. ホームに戻り「Slack 接続テスト」 → 自分の user / team が表示されればAPI疎通OK
3. 「Beacon スキャン開始」 → 「位置情報の使用」「常に許可」→ 「Bluetooth」許可
4. iBeacon を 50cm 以内 → 通知 + Slack に :kinmu: 反映

## 7.3 受け入れ基準チェックリスト (Phase 1 仕様書 §12)

### A. 検知できる

- [ ] 開発者本人の端末で **iBeacon を検知** できる
  - 確認方法: 設定画面で UUID/Major/Minor を入れて「Beacon スキャン開始」 → 直近の生 Beacon カードに UUID が表示される

### B. 3 秒以内にステータス切替

- [ ] 検知から **3 秒以内** に Slack カスタムステータスが `:kinmu:` に切り替わる
  - 確認方法: PC の Slack を表示しっぱなしにして、iBeacon を近づけ → 自分のアイコン横の絵文字が `:kinmu:` に切り替わる時間を計測
  - 内訳目安: Beacon 受信〜JS イベント = 1秒、`fetch` = 1〜2秒、Slack 反映 = 即時

### C. 同日2回目はスキップ

- [ ] 同日2回目以降は Slack API が呼ばれない
  - 確認方法: 設定 > デバッグ > 「本日の打刻」が `YYYY-MM-DD` の状態で再度 iBeacon を近づける
  - 「直近の打刻アクション」が `already today (YYYY-MM-DD)` になる
  - Slack 側のステータスは更新されない (絵文字が消えていない)

### D. iOS で動作

- [ ] iPhone 実機で動作確認済み
- [ ] iOS の「常に許可」で位置情報を許諾済み
- [ ] Bluetooth ON
- [ ] `app.json` の Background Modes (`location`, `bluetooth-central`) が設定済 (Step 1 で確認)

### E. バックグラウンド検知

- [ ] バックグラウンド (アプリ未操作) で **10 回中 5 回以上** 検知できる
  - 確認方法: Slack を別端末で見ながら、検知したい iPhone を background にして iBeacon に近づける × 10
  - 通知が来た / Slack に :kinmu: が出た回数を数える
  - 5 回未満なら RSSI 閾値を緩める or beacon の電波出力を上げる

### F. トークン非コミット

- [ ] Slack トークンがソースコード/git に含まれない
  - 確認方法: `git grep -n 'xoxp-' || echo OK` で何も出ないこと
  - 確認方法: `cat .gitignore | grep -E '\.env'` で `.env` / `.env.*` 除外を確認
  - 確認方法: `git log -p -- .env` で過去にコミットされた履歴がないこと

## 7.4 統合シナリオ (動作確認の流れ)

### 1. 完全クリア状態から

```bash
# シミュレータ / 実機からアプリを削除してインストールし直す
npx expo run:ios --device
```

1. アプリ起動 → ホーム画面、Slack トークンが「未設定」、Beacon スキャン状態「停止中」
2. 「Slack 接続テスト」 → 「未設定」アラート
3. 設定画面で `xoxp-` を貼り付け → 保存
4. UUID / Major / Minor / RSSI 閾値を入力 → 保存
5. ホームに戻る → トークン欄がマスク表示
6. 「Slack 接続テスト」 → `auth.test` カードに自分の user/team が表示
7. 「Beacon スキャン開始」 → 位置情報ダイアログ「常に許可」 → 「稼働中」
8. iBeacon を近づける → 通知 + Slack ステータス更新

### 2. アプリ再起動シナリオ

1. 一度完全終了 (swipe kill)
2. アプリを再度開く → ホームの「Beacon スキャン状態 = 稼働中」になる (autoStart)
3. iBeacon を近づける → `already today` 通知 (同日 2回目以降)

### 3. 翌日シナリオ (シミュレーション)

1. 設定 > デバッグ > 「本日の打刻記録をクリア」
2. iBeacon を近づける → 新規打刻

## 7.5 よくあるエラー集

### iOS ビルド系 (prebuild / pod install / Xcode)

| 症状 | 対処 |
|---|---|
| `pod install` で `Unable to find a specification for 'RCT-Folly' depended upon by 'react-native-beacon-radar'` | `app.json` の `expo.newArchEnabled` を `false` に。ライブラリが新アーキ未対応 (Step 1.3 参照)。`rm -rf ios && npx expo prebuild --clean` |
| `xcrun: error: SDK "iphoneos" cannot be located` / `configure: error: C compiler cannot create executables` | `xcode-select` がフル版 Xcode を指していない。`sudo xcode-select -s /Applications/Xcode.app/Contents/Developer` (§7.1.A-1) |
| `Insufficient space available. Requires X GB` (Xcode Components 画面) | ディスク容量不足。`rm -rf ~/Library/Developer/Xcode/DerivedData` と `rm -rf ~/Library/Developer/Xcode/iOS\ DeviceSupport` で 7 GB 以上空く (§7.1.A-2) |
| `Cannot create a iOS App Development provisioning profile ... do not support the Push Notifications capability` | Personal Team は Push 非対応。`app.json` の `plugins` から `expo-notifications` 除外 + entitlements の `aps-environment` 削除 (§7.1.A-6) |
| `Could not find a profile matching ...` / `No profiles for ... were found` | Xcode の `Signing & Capabilities` で Team を Personal Team に設定 (§7.1.A-6)。Bundle ID 重複なら一意化 |
| Xcode プロジェクトが `Phase1.xcworkspace` 等の意図しない名前で生成された | `expo.name` に日本語が混じっている。ASCII 名 (例: `ShusshaBancho`) に直し、ホーム画面表示は `ios.infoPlist.CFBundleDisplayName` で上書き (Step 1.3) |
| `Unexpected devicectl JSON version output` の警告 | Xcode 26 系の devicectl と Expo CLI のフォーマット差。多くは無害。実機が認識されないなら下行を試す |
| `No iOS devices available in Simulator.app` (`--device` 指定なのに) | iPhone 未接続 / 信頼未承認 / デベロッパモード未 ON / ロック中。`xcrun devicectl list devices` で見えるか確認 (§7.1.A-7) |
| `iOS X.Y is not installed` (Xcode 上部) | プラットフォームサポート未インストール。Xcode Components 画面で対象 iOS を `Get` (§7.1.A-2) |
| `BeaconRadar` モジュールが見つからない (実機実行時) | `npx expo prebuild --clean` してから `expo run:ios --device` で native ビルドし直す |
| iPhone で「信頼されていないデベロッパ」アイコンがグレーアウト | iPhone 設定 > 一般 > VPN とデバイス管理 → Apple Development の項を「信頼」(§7.1.A-8) |
| 7 日後にアプリが起動しなくなった | Personal Team の有効期限切れ。`expo run:ios --device` で再ビルドして再インストール |

### 動作系 (アプリ起動後)

| 症状 | 対処 |
|---|---|
| `:kinmu:` が `:question:` っぽい | カスタム絵文字未登録。Slack ワークスペース管理で `:kinmu:` を入れる |
| `invalid_auth` がアプリで連発、curl では成功 | アプリ側の SecureStore に古いトークンが残っている。設定で「削除」→ 貼り直し |
| Background で全く動かない | iOS 設定 > 出社番長 > 位置情報 = 「常に」、Bluetooth ON、機内モード OFF |
| Android の常駐通知が出ない | `useForegroundService: true` が動かない端末。OS バージョンを確認、`POST_NOTIFICATIONS` 許可済か |
| `getJstDateString()` が翌日になる | 端末時計が狂っている。「設定 > 日付と時刻 > 自動」に |

## 7.7 Phase 2 への引き継ぎメモ (将来用)

仕様書 §13 に書かれている方針を **コードレベルで再確認**:

- `slackClient.setStatus(token, ...)` → `token` を引数で受け取る形は維持されている (✅)

  - Phase 2 では OAuth 経由で取った token を同じ SecureStore に保存する形に拡張
