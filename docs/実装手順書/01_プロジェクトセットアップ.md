# Step 1: プロジェクトセットアップ

Expo + React Native + TypeScript の Dev Client プロジェクトをリポジトリルートに作り、
Phase 1 で必要な依存とパーミッション宣言を投入する。

## 完了条件

- [ ] `npm install` が成功する
- [ ] `npx expo-doctor` がすべての check に PASS する
- [ ] `npx tsc --noEmit` がエラーなしで終わる
- [ ] `App.tsx` を起点に Home / Settings 2 画面のスタックが立ち上がる骨組みがある

## 1.1 Expo プロジェクト初期化

リポジトリルートに既存ファイル（`README.md`, `docs/`, `.git`）があるため、`create-expo-app`
は一時ディレクトリで生成 → ルートにマージする。

```bash
# 一時ディレクトリで生成
mkdir -p /tmp/shussha-init
cd /tmp/shussha-init
npx --yes create-expo-app@latest shussha-bancho --template blank-typescript --no-install

# .git を除いてリポジトリルートにコピー
rsync -av --exclude='.git' /tmp/shussha-init/shussha-bancho/ /Users/tagami/shussha-bancho/
rm -rf /tmp/shussha-init
```

> パス（`/Users/tagami/shussha-bancho/`）は自身の環境に合わせて読み替えること。

## 1.2 `.gitignore` の強化

デフォルトの `.gitignore` は `.env*.local` のみ無視するが、Phase 1 では
**`.env` 全般を一切コミットしない** 運用にする。

```diff
- # local env files
- .env*.local
+ # env files (Slackトークンなど秘密情報をコミットしない)
+ # .env.example のみコミット可
+ .env
+ .env.*
+ !.env.example
```

`.env.example` は **Slack トークンや Beacon 設定値の "サンプル"** をコミットしておくためのファイル。
実際の値を書く `.env` は決してコミットしない。

```env
# .env.example
SLACK_USER_TOKEN=
BEACON_UUID=
BEACON_MAJOR=
BEACON_MINOR=
RSSI_THRESHOLD=-75
```

## 1.3 `app.json` のカスタマイズ

Phase 1 では以下を反映する:

- `name` / `slug` / `bundleIdentifier` / `package` の社内識別子化
- iOS / Android の **位置情報・Bluetooth・通知** 関連のパーミッション宣言
- iOS の `UIBackgroundModes`（`location`, `bluetooth-central`, `fetch`）
- 必要な **config plugin** の登録
  - `expo-secure-store` (Keychain / Keystore)
  - `expo-location`（許諾文言・バックグラウンド有効化）
  - `expo-task-manager`
  - `react-native-beacon-radar`
  - `expo-build-properties`（iOS deploymentTarget / Android minSdkVersion）

> `bundleIdentifier` / `package` は `jp.co.fusic.shusshabancho.phase1` を仮置きしている。
> 自社の命名規則に合わせて変更可（変更時は EAS / Apple Developer 側の登録もやり直す）。

実ファイルは `app.json` を参照。

### 重要な決定事項 (詰まりポイント)

Phase 1 PoC を **無料 Apple ID (Personal Team)** で動かすために、以下 4 点は強制:

| 項目 | 値 | 理由 |
|---|---|---|
| `expo.newArchEnabled` | **`false`** | `react-native-beacon-radar 0.3.2` の podspec が新アーキ時に `RCT-Folly` / `React-Codegen` という旧名 Pod を要求し、Expo SDK 54 (RN 0.81) で `pod install` が失敗するため |
| `expo.name` | **ASCII のみ** (例: `ShusshaBancho`) | Expo は `expo.name` から Latin 文字のみ抽出して Xcode プロジェクト名にする。日本語混じりだと意図しない名前 (例: 括弧前後だけ拾って `Phase1`) になる |
| `expo.ios.infoPlist.CFBundleDisplayName` | 日本語 OK (例: `出社番長 (Phase1)`) | iPhone ホーム画面の表示名はこちらで上書き |
| `plugins` から `expo-notifications` を **除外** | プラグイン未登録 | Personal Team は **Push Notifications capability 非対応**。`expo-notifications` は無条件で `aps-environment` entitlement を追加する設計のため、provisioning profile 作成が `Cannot create a iOS App Development provisioning profile ... do not support the Push Notifications capability` で必ず失敗する |

> **ローカル通知 (`Notifications.scheduleNotificationAsync` 等) は autolinking で動作する**ので、`npm` 依存からは外さなくて良い。`plugins` からだけ外す。
> Apple Developer Program ($99/年) に加入したら `plugins` に戻す。

## 1.4 依存パッケージの導入

```bash
npm install
npx expo install \
  expo-dev-client \
  expo-secure-store \
  expo-task-manager \
  expo-location \
  expo-notifications \
  expo-build-properties \
  @react-native-async-storage/async-storage \
  react-native-beacon-radar \
  @react-navigation/native \
  @react-navigation/native-stack \
  react-native-screens \
  react-native-safe-area-context
```

`npx expo install` は `package.json` の `expo` バージョンに合うパッケージバージョンを
自動で選定する。`npm install` 単独で入れると SDK 互換が崩れることがあるので **必ず `expo install`** を使う。

## 1.5 `tsconfig.json` にパスエイリアス

```jsonc
{
  "extends": "expo/tsconfig.base",
  "compilerOptions": {
    "strict": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  },
  "include": [
    "**/*.ts",
    "**/*.tsx",
    ".expo/types/**/*.ts",
    "expo-env.d.ts"
  ]
}
```

Expo SDK 50 以降は **Metro が tsconfig の path alias を自動で解決する** ので
`babel-plugin-module-resolver` のような追加設定は不要。

## 1.6 ディレクトリ構成

以下を作成する:

```
.
├── App.tsx                     ← React Navigation の Root
├── index.ts                    ← Expo 既定 (registerRootComponent)
├── app.json
├── package.json
├── tsconfig.json
├── .env.example
├── assets/                     ← Expo 既定アイコン群
├── docs/
│   ├── 仕様書/                 ← 既存
│   └── 実装手順書/             ← 本手順書
└── src/
    ├── config/
    │   └── constants.ts        ← :kinmu: 定数、Storage キーなど
    └── screens/
        ├── HomeScreen.tsx      ← ホーム
        └── SettingsScreen.tsx  ← 設定 (Step 2 で本実装)
```

`src/config/constants.ts` には Phase 2 でバックエンドへ移してもパスを変えるだけで済むよう、
**画面・モジュール側からは必ずこの定数経由で参照** することを徹底する。

## トラブルシュート

| 症状                                                              | 原因と対処                                                                                                                          |
| ----------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `expo-doctor` が `react-native-beacon-radar` で SDK 警告           | パッケージは Expo 管理外。`npx expo install` で入れていれば doctor は警告止まりで OK。                                              |
| `tsc` が `Cannot find module '@/screens/...'`                      | `tsconfig.json` の `baseUrl` と `paths`、`include` の指定漏れ。Step 1.5 の通り設定。                                                |
| Metro が `@/...` を解決しない                                      | Expo SDK 50 未満。本リポジトリは SDK 54 なので不要だが、もし発生したら `metro.config.js` で `resolver.alias` を設定する。           |
| `npm audit` が脆弱性警告を出す                                     | 開発依存（`@xmldom` 等）の警告。Phase 1 PoC では一旦無視可。Phase 2 移行時に `npm audit fix` で対応する。                           |
| `pod install` で `Unable to find a specification for 'RCT-Folly'` | `react-native-beacon-radar` が新アーキ時に旧名 Pod を要求するため。`app.json` で `"newArchEnabled": false` にする (1.3 の表参照)。  |
| Xcode プロジェクト名が想定と違う (例: `Phase1.xcworkspace`)        | `expo.name` に日本語が含まれる。ASCII のみに変更し、ホーム画面表示名は `expo.ios.infoPlist.CFBundleDisplayName` で上書き (1.3)。     |
| `Cannot create a iOS App Development provisioning profile ... do not support the Push Notifications capability` | Personal Team は Push 非対応。`app.json` の `plugins` から `expo-notifications` を外し、`ios/<Project>/<Project>.entitlements` の `aps-environment` を消す (1.3)。 |
