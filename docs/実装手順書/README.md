# 出社番長 Phase 1 実装手順書

`docs/仕様書/Phase1_Slackステータス連携.md` に対応する **実装手順書（ステップ書）** です。
上から順に実施することで、開発者1名分の最小 PoC が手元の端末で動く状態になります。

> 本手順書は **このリポジトリのルート直下に Expo プロジェクトを構築する** 前提で書かれています。
> Phase 2 以降は別ディレクトリ／別ブランチで進めるか、当リポジトリを Phase 1 専用に保つかを別途決めてください。

## 前提

- macOS（iOS 端末で動作確認するため Xcode が必須）
- Node.js 20 以上（推奨: 24 系。現環境は Node 24.7.0 で動作確認済み）
- npm 10 以上
- Apple Developer アカウント（実機で iBeacon 検知を試すため。無料アカウントでも可）
- Expo Account（EAS Build を使う場合）
- Slack ワークスペース管理者権限（カスタム App 作成のため）
- iBeacon 端末 1 台（仕様書に従い MM-BLEBC3 を想定）

## 進め方の原則

- **1 ステップずつ完了させて commit** する。途中で挫折しても次のステップから再開できるように。
- **Slack トークンは絶対にコミットしない**。`.env` と `.env.*` は `.gitignore` 済み（`.env.example` のみコミット対象）。
- 検証は **必ず実機**。iBeacon は iOS Simulator / Android Emulator では検知できない。
- ステップ末の "完了条件" を 1 つでも満たさない場合は次に進まない。
