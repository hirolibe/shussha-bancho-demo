# オフィス個室ワークスペース在室管理システム 開発計画書

## 1. プロジェクト概要

### 1.1 目的
社内に設置されている個人用ワークスペース(個室)の利用状況を、社員が手元のスマホやSlackからリアルタイムで確認できるようにする。さらに、利用統計の取得・自動Slackステータス連携・利用ログ取得など、ワークスタイル改善に役立つデータと体験を提供する。

### 1.2 システムの基本コンセプト
- **「やりたい人だけ」が使えるオプトイン型**(全社強制ではない)
- BLEビーコンを各個室に設置 → 社員のスマホアプリが電波を検知 → サーバーで状態管理 → Slack & ダッシュボードに反映
- 軽量・低コスト・運用負荷小

### 1.3 想定ユーザー
- ワークスペースを利用する社員(20〜50名程度)
- 利用状況を可視化したい総務・経営層

## 2. システム構成

### 2.1 全体アーキテクチャ

```
┌─────────────────┐
│ 各個室            │
│  [BLEビーコン]    │  ← iBeacon電波を発信
└─────────────────┘
         ↓ BLE
┌─────────────────┐
│ 社員のスマホ       │
│ (Expoアプリ)      │  ← ビーコン検知 → API送信
└─────────────────┘
         ↓ HTTPS
┌─────────────────┐
│ バックエンドAPI    │
│ (Firebase/AWS)   │  ← 状態管理 / ロジック処理
└─────────────────┘
         ↓
   ┌─────┴─────┐
   ↓           ↓
[Slack API]  [ダッシュボード]
            (Next.js)
```

### 2.2 技術スタック

| レイヤー | 技術 | 理由 |
|---|---|---|
| ビーコン | iBeacon対応BLEビーコン(市販) | 自作はROIが悪い、市販で十分 |
| モバイルアプリ | React Native + Expo (Development Build) + react-native-beacons-manager | 既存ナレッジあり、iOS/Android両対応 |
| バックエンド | Firebase (Firestore + Cloud Functions + Authentication) | サーバーレスで運用負荷小、無料枠で開始可能 |
| ダッシュボード | Next.js + Tailwind CSS | 静的ホスティングしやすい、開発速度 |
| 認証 | Google認証 (社内ドメイン制限) | 社員のみアクセス可能にする |
| Slack連携 | Slack Bolt SDK + Web API | ステータス変更・スラッシュコマンド対応 |
| デプロイ | Vercel (ダッシュボード) / Firebase Hosting | 無料枠で開始可能 |

## 4. BLEビーコンの選定

### 4.1 選定基準
- **iBeacon対応**(iOSのバックグラウンド検知に必須)
- **電池寿命2年以上**(交換頻度を抑える)
- **国内技適マーク取得済**(電波法対応)
- **複数台導入時の単価が抑えられる**

### 4.2 推奨製品(プロトタイプ用 → 本番運用)

#### 🥇 第1候補: サンワサプライ MM-BLEBC4(国内品 / 推奨)
- **価格**: 3個セットで1万円前後
- **特徴**: iBeacon・Eddystone両対応、大容量バッテリー、技適取得済み
- **電池寿命**: 数ヶ月〜3年(設定による)
- **設定**: iOS/Android用の設定アプリで電波強度・送信間隔を変更可能
- **送信範囲**: 1〜100mまで8段階調整可能(個室では弱めに設定)
- **国内サポートあり、説明書も日本語で安心**

#### 🥈 第2候補: アプリックス USBビーコン2 USB02W
- **価格**: 1個 数千円〜
- **特徴**: USB給電のため電池切れ心配なし、Bluetooth 5.0対応
- **メリット**: 個室にUSBコンセントがあれば電池交換不要

#### 🥉 第3候補: Feasycom FSC-BP108(海外品 / 安価)
- **価格**: 1個 1,500〜3,000円
- **特徴**: Amazonで入手可、防水、Bluetooth 5.1
- **注意**: 技適マークの確認必須(個人利用なら可だが業務利用は要確認)

### 4.3 選定方針
- **MVPプロトタイプ段階**: サンワサプライMM-BLEBC4を3個購入(1万円程度)
- **本番展開時**: 個室ごとに1個 + 予備1個
- **既製品で十分**: ESP32などで自作する選択肢もあるが、技適取得・耐久性・運用コストを考えると市販品一択

### 4.4 自作 vs 既製品の比較

| 項目 | 既製品(推奨) | 自作(ESP32等) |
|---|---|---|
| 初期コスト | 1個 2,000〜3,000円 | 1個 1,500円程度 |
| 開発工数 | 0(設定のみ) | 数日〜 |
| 技適 | 取得済 | 個人利用範囲なら可、業務は要確認 |
| 安定性 | ◎ | △(電源管理・故障対応が必要) |
| 結論 | **既製品を選ぶべき** | 学習目的なら可 |

## 5. データモデル

### 5.1 Firestore スキーマ

```
users/{userId}
  - email: string
  - displayName: string
  - notificationPreferences: {...}
  - createdAt: timestamp

rooms/{roomId}
  - name: string ("個室A", "集中ブース1" など)
  - beaconUuid: string
  - beaconMajor: number
  - beaconMinor: number
  - rssiThreshold: number (デフォルト -65)
  - currentStatus: "available" | "occupied"
  - currentUserId: string (optional)
  - lastUpdated: timestamp

sessions/{sessionId}
  - roomId: string
  - userId: string
  - startTime: timestamp
  - endTime: timestamp (optional)
  - duration: number (秒)
  - autoDetected: boolean
```

### 5.2 入退室判定ロジック

- **入室**: RSSI が閾値以上(例: -65dBm 以上)で 5秒以上継続検知
- **退室**: 検知が 30秒間途絶えたら退室と判定
- **誤検知対策**: 入退室の連続発生を防ぐため、状態変化後は10秒間ロック

## 8. リスクと対策

### 8.1 技術リスク

| リスク | 影響 | 対策 |
|---|---|---|
| iOSバックグラウンド検知制限 | アプリが閉じてると検知不可 | iBeaconフォーマット限定、`startMonitoring`を活用 |
| ビーコン電波の隣室漏れ | 誤検知が起きる | RSSI閾値を厳しく設定、必要に応じて出力強度を弱く |
| アプリ未インストール者は検知不可 | 統計が偏る | Phase 3で人感センサー併用を検討 |
| ビーコン電池切れ | 静かに検知不能になる | 死活監視ジョブで「最近検知されてないビーコン」をアラート |
| スマホを置いて離席 | ずっと使用中になる | 30秒以上RSSI変化なし+加速度なし、で離席判定(将来) |

### 8.2 運用リスク

| リスク | 影響 | 対策 |
|---|---|---|
| 社員の理解不足 | 利用率が上がらない | 社内告知・デモ実施、利用者の声をフィードバックに |
| プライバシー懸念 | 利用拒否 | オプトイン徹底、データの利用目的を明示、本人のみ閲覧可能なデータも作る |
| Slack運用との相性 | 通知うざがられる | 通知頻度をユーザー設定可能に、デフォルトは控えめ |

### 8.3 プライバシー設計

- **オプトイン**: アプリインストール=同意、と明確化
- **データ最小化**: 必要なデータのみ保持、退室から3ヶ月で個人特定情報は匿名化
- **可視化レベル**: 「誰が使ってるか」を見せるかは個人設定で選べる(デフォルトは「使用中/空きのみ」)
- **退会機能**: アプリからいつでも退会・データ削除可能
